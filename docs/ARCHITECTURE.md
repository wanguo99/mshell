# MShell 架构文档

## 架构概览

MShell采用分层架构设计，实现了模块解耦和职责分离。

```
┌─────────────────────────────────────────────────────────────┐
```

## 核心组件说明

### 1. 数据模型层 (models/)

**ConnectionConfig**: 连接配置基类
- `SSHConnectionConfig`: SSH连接配置
- `SerialConnectionConfig`: 串口连接配置
- `create_connection_config()`: 工厂方法

**Session**: 会话模型
- 封装连接对象、终端widget、配置信息
- 提供统一的会话管理接口

### 2. 服务层 (services/)

**ConfigService**: 配置服务
- 封装ConfigManager，提供统一接口
- 返回强类型的ConnectionConfig对象
- 处理配置的增删改查

### 3. 业务逻辑层 (core/managers/)

**ConnectionManager**: 连接管理器
- 管理所有活动会话
- 创建SSH/串口连接
- 处理连接生命周期
- 使用信号机制与UI层通信

### 4. 表现层 (ui/)

**MainWindow**: 主窗口
- 集成所有UI组件
- 响应用户操作
- 调用业务逻辑层服务

**Sidebar**: 侧边栏组件
- 显示已保存的连接
- 提供新建连接按钮
- 发送信号通知主窗口

**WelcomePage**: 欢迎页组件
- 显示欢迎信息
- 独立可复用

## 信号流转

```
用户操作 → UI组件发送信号 → MainWindow处理 → 调用业务逻辑层
                                              ↓
                                    ConnectionManager创建连接
                                              ↓
                                    发送connection_created信号
                                              ↓
                                    MainWindow添加标签页
```

## 依赖关系

- **UI层** 依赖 **业务逻辑层** 和 **服务层**
- **业务逻辑层** 依赖 **数据模型层** 和 **基础设施层**
- **服务层** 依赖 **数据模型层** 和 **基础设施层**
- **数据模型层** 无依赖（纯数据对象）
- **基础设施层** 无业务依赖

## 优势

1. **高内聚低耦合**: 每个模块职责单一，依赖关系清晰
2. **易于测试**: 各层可独立测试，便于编写单元测试
3. **易于扩展**: 新增连接类型只需扩展模型和管理器
4. **代码复用**: UI组件可在不同场景复用
5. **维护性强**: 修改某层不影响其他层
6. **可读性好**: 代码组织清晰，易于理解

## 性能优化

1. **终端渲染优化**
   - 刷新率从20 FPS降至10 FPS
   - 批量处理数据，减少重绘次数
   - 历史行渲染从1000行减至500行

2. **焦点管理优化**
   - 所有UI元素设置NoFocus
   - Tab键专用于终端命令补齐
   - 焦点始终保持在终端窗口

## 迁移说明

旧版本代码已备份为 `main_old.py`，如需回退：
```bash
mv main.py main_new.py
mv main_old.py main.py
```
